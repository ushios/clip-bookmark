# Feature Specification: Twitch Bookmark Trigger (Twitch ブックマーク・トリガー機能)

**Feature Branch**: `001-twitch-bookmark`

**Created**: 2026-07-04

**Status**: Draft

**Input**: User description: "URLごとに何時何分かを記録していくのがいいと思っていて、ショートカットを作るか、決まったワードを自分がコメントに入れるかすると、そのタイミングを後から確認できる、また、その時間の動画に飛べるとかがいいと思ってます。どうでしょうか"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - キーボードショートカットによるブックマーク記録 (Priority: P1)

ユーザーがTwitchの配信またはVODの視聴中に、設定されたキーボードショートカット（例: Alt+Shift+B）を押すと、現在の再生位置（VODの場合は再生時間、ライブの場合は配信開始からの経過時間）がその動画/配信のURLと共に即座に保存され、ユーザーに記録完了が通知されます。

**Why this priority**: 配信中の「ここだ！」と思った瞬間を、画面操作や視線移動をすることなく最も素早く直感的に記録するために極めて重要です。

**Independent Test**: Twitchの配信またはVODページを開き、ショートカットキーを押して、画面上に「ブックマーク保存完了」の通知が表示されること。また、拡張機能のポップアップを開いて、正しい再生時間のブックマークが追加されていることを確認します。

**Acceptance Scenarios**:

1. **Given** ユーザーがVOD（`https://www.twitch.tv/videos/12345`）の 01:23:45 の位置を再生中であり、
   **When** キーボードショートカット `Alt+Shift+B` を入力したとき、
   **Then** 「01:23:45 をブックマークしました」という通知が表示され、データが保存されること。
2. **Given** ユーザーがライブ配信（`https://www.twitch.tv/streamer`）を視聴中で、配信開始から 2時間30分15秒 が経過しているとき、
   **When** キーボードショートカット `Alt+Shift+B` を入力したとき、
   **Then** 「02:30:15 をブックマークしました」という通知が表示され、データが保存されること。

---

### User Story 2 - チャットコメントによる自動ブックマーク記録 (Priority: P1)

ユーザーがTwitchのチャット入力欄に特定のキーワード（例: `!bm` や `!bookmark`）を入力して送信すると、拡張機能がこれを検知して、その送信時点の再生位置をブックマークとして自動記録します。また、ユーザーは拡張機能のポップアップUI（または設定画面）から、トリガーとなるキーワードを追加・変更・削除することができます。

**Why this priority**: チャットでのリアクションや発言と同時にブックマークを記録できるため、キーボードショートカットとは異なる快適な体験を提供します。

**Independent Test**: 拡張機能の設定で新しいキーワード（例: `!clip`）を追加し、Twitchの配信ページでチャット欄にそのキーワードを入力して送信します。拡張機能のポップアップを開き、送信した時点の再生時間がブックマークとしてリストに追加されていることを確認します。

**Acceptance Scenarios**:

1. **Given** ユーザーがライブ配信のチャット欄を開いており、
   **When** チャットメッセージとして `!bm`（または設定されたキーワード）を送信したとき、
   **Then** 拡張機能がそれを検知し、その瞬間の経過時間をブックマークとして自動保存すること。
2. **Given** ユーザーがカスタムトリガーワードを追加したいとき、
   **When** ポップアップ内の設定画面で「ここ！」というワードを登録したとき、
   **Then** チャットで「ここ！」と送信した際にもブックマークが保存されること。

---

### User Story 3 - ポップアップUIでの履歴確認と該当時間へのジャンプ (Priority: P1)

ユーザーが拡張機能のポップアップUIを開くと、保存されたブックマークの一覧が新しい順に表示されます。各ブックマークには配信タイトル（または配信者名）、記録日時、および経過時間が表示され、クリックすると該当する時間のVODページが新しいタブで開きます。

**Why this priority**: 記録したブックマークを後から見返し、ワンクリックでその動画の該当するシーンにジャンプしてクリップ作成などの作業を行うためのコア機能です。

**Independent Test**: ポップアップUIを開き、リストにあるブックマーク項目をクリックし、新しいタブでTwitchの動画ページが開き、記録された正確な時間（`?t=...`）から再生が開始されることを確認します。また、不要なブックマークをリストから削除できることをテストします。

**Acceptance Scenarios**:

1. **Given** ユーザーが拡張機能のポップアップUIを開いたとき、
   **When** 過去に保存した「01:23:45」のブックマークをクリックしたとき、
   **Then** 新しいブラウザタブで `https://www.twitch.tv/videos/12345?t=1h23m45s` が開くこと。
2. **Given** ユーザーが特定のブックマーク履歴を削除したいとき、
   **When** リスト項目の削除ボタン（ごみ箱アイコンなど）をクリックしたとき、
   **Then** その項目がリストから消去され、`chrome.storage` からも削除されること。

---

### Edge Cases

- **ライブ配信中のブックマークとVODのズレ**: ライブ配信中に記録した「経過時間」と、配信終了後に生成されるVOD（アーカイブ）の「再生時間」に若干のズレ（配信開始前の待機画面や配信トラブルによる切断など）が生じる可能性があります。
  - **対応策**: ライブ配信のブックマークには配信開始日時と記録日時の差分を保存し、VODへの遷移時にはできる限り近い時間を推測して遷移するか、多少のズレが発生し得ることを「ライブ記録」として明示します。
- **プレイヤーの未ロード状態**: プレイヤーのDOM要素がロードされる前にショートカットキーが押された場合。
  - **対応策**: プレイヤー要素が検出できない場合は、ブックマーク保存失敗の通知を表示し、動作しない理由を伝えます。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 拡張機能は、キーボードショートカット（Alt+Shift+B 等、ブラウザ標準の `chrome://extensions/shortcuts` で変更可能）の押下イベントを検知し、アクティブなTwitch視聴ページのプレイヤーから現在時間を取得できなければならない。
- **FR-002**: 拡張機能は、ユーザー自身が送信したチャットメッセージを監視し、登録されたトリガーワード（デフォルト: `!bm`）を検知した際に、自動で現在時間を記録しなければならない。
- **FR-003**: 記録するデータには、「配信者名/チャンネル名」「配信タイトル（取得可能な場合）」「記録した絶対日時」「配信開始からの経過時間（またはVODの再生位置）」「VODのURL（またはベースとなるチャンネルURL）」を含めなければならない。
- **FR-004**: 記録されたデータは、ブラウザを閉じても保持されるよう `chrome.storage.local`（または `chrome.storage.sync`）に永続化されなければならない。
- **FR-005**: 拡張機能のポップアップUIは、保存されたブックマークを最新順に一覧表示し、各項目から該当時間のVOD（`?t=XhYmZs` 形式）へ遷移するリンク提供しなければならない。
- **FR-006**: ポップアップUIにおいて、任意のブックマークを個別に、または全件一括で削除できる機能を提供しなければならない。
- **FR-007**: ポップアップUIまたは設定画面において、チャット監視用のトリガーワードをユーザーが自由に追加・削除できる管理機能を提供しなければならない。

### Key Entities

- **Bookmark (ブックマーク)**:
  - `id`: ユニークなID（UUIDまたはタイムスタンプ）
  - `channelName`: 配信者のチャンネル名（例: `streamer_name`）
  - `title`: 配信またはVODのタイトル
  - `videoUrl`: VODのベースURL（例: `https://www.twitch.tv/videos/12345`。ライブの場合はチャンネルURL）
  - `timestamp`: ブックマークを記録した絶対日時 (ISO 8601形式)
  - `relativeTime`: 配信開始からの経過時間、またはVOD内の再生位置（秒数、例: `5025`）
  - `isLive`: ライブ配信中に記録されたかどうかのフラグ

- **Settings (設定)**:
  - `triggerWords`: トリガーとなるキーワードの配列（例: `['!bm', '!bookmark', 'ここ！']`）

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーがキーボードショートカットを押す、またはチャットでトリガーワードを送信してから、ブックマークが正常に保存されて「保存完了」の画面通知が表示されるまでの処理時間が **0.3秒以内** であること。
- **SC-002**: ポップアップ内のブックマークリンクをクリックした際、新規タブが立ち上がり、記録した時間と **誤差3秒以内** の正しい再生位置から動画が開始されること。
- **SC-003**: ポップアップUI起動時、保存件数が100件の場合でも、リスト表示完了までの時間が **0.2秒以下** であること。

## Assumptions

- ユーザーはGoogle Chromeブラウザを使用し、Twitchにログインして配信またはVODを視聴していることを前提とします。
- VODへのリンク遷移（`?t=...`）はTwitch公式のタイムスタンプ機能に依存します。
- ライブ配信中の配信経過時間は、Twitchのプレイヤー内に表示される配信時間、またはAPIから取得できる配信開始時刻を基準に算出します。
