# 手動でのアップデート・ストア公開手順

本ドキュメントでは、GitHub Actions を使用せず、**「手動で拡張機能をビルドして Chrome ウェブストアにアップデート（アップロード）する手順」** について解説します。
複雑な Google Cloud や OAuth の設定は一切不要で、誰でも 1 分程度で完了できます。

---

## 🛠️ アップデート手順（全 5 ステップ）

コードの修正や機能追加が終わり、ストアに最新版を反映させたくなったら、以下の手順を実行します。

### ステップ 1：バージョン番号の更新
Chrome ウェブストアでは、古いバージョンと同じ番号のパッケージはアップロードできません。必ずバージョン番号を更新します。

1.  [src/manifest.json](file:///Users/shugo/Develops/ushios/clip-bookmark/src/manifest.json) を開きます。
2.  `"version": "1.0.0"` の部分を、新バージョン（例：`"1.0.1"` や `"1.1.0"`）に書き換えて保存します。

---

### ステップ 2：ビルドの実行
TypeScript や CSS のコードを、ブラウザが読み込めるJavaScriptにコンパイルします。

1.  プロジェクトのルートディレクトリで、ターミナルを開きます。
2.  以下のコマンドを実行します：
    ```bash
    npm run build
    ```
    *※これにより、`dist/` ディレクトリ配下に最新のコードが出力されます。*

---

### ステップ 3：ZIP パッケージの作成
アップロード用の ZIP ファイルを作成します。

> [!IMPORTANT]
> **注意**: `dist` フォルダ自体を圧縮するのではなく、**「`dist` フォルダの内部にあるファイル群」** を直接ZIPに詰める必要があります。

ターミナルで以下のコマンドを一発実行するのが最も確実で簡単です（ルートフォルダ直下に `clip-bookmark.zip` が自動作成・上書きされます）。

```bash
cd dist && zip -r ../clip-bookmark.zip * && cd ..
```

---

### ステップ 4：デベロッパーコンソールへのアップロード
1.  [Chrome ウェブストア デベロッパーコンソール](https://chrome.google.com/webstore/devconsole/) にアクセスします。
2.  公開中のアイテム（Clip Bookmark）をクリックして管理画面に入ります。
3.  左側メニューの **「パッケージ」** をクリックします。
4.  **「新しいパッケージをアップロード」** ボタンをクリックします。
5.  ステップ 3 で作成した **`clip-bookmark.zip`** を画面にドラッグ＆ドロップします。

---

### ステップ 5：審査の提出
1.  アップロードが成功したら、画面右上にある **「審査に提出」** ボタンをクリックします。
2.  審査状況が「審査中」になれば完了です。通常数日〜1週間程度でユーザーに自動アップデートが配信されます。
